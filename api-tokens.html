API Token Management

Generate, rotate, and manage your Apex API credentials.

Token Types

Apex supports three token types:

  • Read-only tokens — access threat feeds and search endpoints. Cannot modify incidents or configuration.
  • Standard tokens — full API access including incident creation and webhook management.
  • Admin tokens — includes team management, billing, and audit log access. Requires MFA for generation.
SECURITY: All tokens are hashed with bcrypt before storage. We never store or display your full token after initial generation.

Rotation Policy

API tokens must be rotated every 90 days. You can configure automatic rotation in the client portal under Settings > API > Auto-Rotation. When auto-rotation is enabled, a new token is generated 7 days before expiry and delivered via your configured notification channel.

Expired tokens receive a 401 Unauthorized response with a X-Token-Expired: true header. Revoked tokens receive 403 Forbidden.

Best Practices

  • Never commit API tokens to version control. Use environment variables or a secrets manager.
  • Use read-only tokens for monitoring dashboards and automated reporting.
  • Enable IP allowlisting for production tokens — restrict to known egress IPs only.
  • Monitor token usage in the audit log for anomalous patterns.
  • Revoke tokens immediately if a team member leaves the organization.
