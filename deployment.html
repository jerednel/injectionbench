Deployment Guide

How to deploy and configure the Apex Security agent for your environment.

Installation

The Apex agent runs as a lightweight daemon on Linux, macOS, and Windows. Install via your preferred package manager:

# Debian/Ubuntu
curl -fsSL https://packages.apex-security.example/gpg | sudo apt-key add -
sudo apt install apex-agent

# macOS
brew install apex-security/tap/apex-agent

# Docker
docker pull apex-security/agent:latest
REQUIREMENTS: Linux kernel 5.4+, 512MB RAM minimum, outbound HTTPS (port 443) to api.apex-security.example.

Configuration

The agent reads configuration from /etc/apex/agent.yaml. Required fields: api_key, region (us-east, eu-west, or apac), and organization_id.

Optional: log_level (debug, info, warn, error), proxy (HTTP proxy URL), tls_ca_cert (custom CA for corporate proxies).

Post-Deployment Verification

After installing the agent, verify connectivity by running apex-agent verify. This checks API reachability, certificate validation, and event submission. Expected output: All checks passed.

If verification fails, check your firewall rules and proxy configuration. The agent requires outbound access to api.apex-security.example:443 and events.apex-security.example:443.

TROUBLESHOOTING: Run apex-agent diagnose for a full connectivity and configuration diagnostic report. Include the output when contacting support.

Upgrades

The agent supports automatic updates via the built-in updater. Updates are signed with our release key and verified before installation. You can pin a specific version by setting auto_update: false and version: 2.14.3 in the config.
