API Documentation

Reference documentation for the Apex Threat Intelligence API v3.

Authentication

All API requests require a valid API key passed in the X-Apex-Key header. API keys are scoped to specific endpoints and rate limits based on your subscription tier.

Example request:

curl -H "X-Apex-Key: your_api_key_here" \
     -H "Content-Type: application/json" \
     https://api.apex-security.example/v3/threats/recent
RATE LIMITS: Free tier: 100 req/hr. Professional: 10,000 req/hr. Enterprise: unlimited with burst capacity.

Endpoints

GET /v3/threats/recent

Returns the 50 most recent threat indicators. Supports pagination via ?cursor= parameter. Response includes IOCs, MITRE ATT&CK mappings, and confidence scores.

GET /v3/threats/search

Full-text search across threat intelligence database. Supports Boolean operators, IOC type filtering, and date ranges. Maximum 1,000 results per query.

POST /v3/incidents/report

Submit a new incident report. Requires title, severity (1-5), description, and optional iocs[] array. Returns incident ID for tracking.

GET /v3/feeds/{feed_id}

Subscribe to a specific threat feed. Available feeds: ransomware, phishing, c2-infrastructure, vulnerability-exploits, insider-threat.

DEPRECATION: The v2 API will be sunset on June 30, 2026. Please migrate to v3 endpoints. See migration guide in the client portal.

Webhooks

Configure real-time alert delivery via webhooks. Apex supports HMAC-SHA256 signature verification on all webhook payloads. Maximum payload size: 256KB. Retry policy: 3 attempts with exponential backoff.

Webhook events: threat.new, threat.updated, incident.created, incident.resolved, feed.updated.

SDKs & Libraries

Official SDKs available for Python, Go, JavaScript/TypeScript, and Rust. Community SDKs available for Java, C#, and Ruby.

Install: pip install apex-security | npm install @apex-security/sdk | go get apex-security.example/sdk

CHANGELOG: SDK v3.2.0 released March 15, 2026. Adds streaming support for threat feeds and improved retry logic.
