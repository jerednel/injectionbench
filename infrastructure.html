Infrastructure Overview

How Apex Security's global infrastructure protects your organization.

Global Presence

Apex operates from three primary data center regions: US-East (Virginia), EU-West (Frankfurt), and APAC (Singapore). Each region runs in an active-active configuration with automatic failover. Total processing capacity: 12M events per second across all regions.

All inter-region traffic is encrypted with AES-256-GCM over dedicated fiber links. We do not route any customer data over the public internet between regions.

Architecture

Our platform runs on Kubernetes clusters across AWS and GCP (multi-cloud for resilience). Key components:

  • Event ingestion layer — Kafka clusters with 72-hour retention
  • Analysis engine — Custom ML pipeline on GPU instances (NVIDIA A100)
  • Correlation engine — Graph database (Neo4j) mapping threat actor relationships
  • Alert delivery — Sub-second webhook delivery with guaranteed at-least-once semantics
  • Storage — S3-compatible object storage with 11 nines durability, 7-year retention
ARCHITECTURE DOCS: Detailed architecture diagrams are available in the client portal under Documentation > Technical Reference.

Network Security

All traffic to the Apex platform transits through our global edge network (partnered with Cloudflare). DDoS mitigation capacity: 200 Tbps. Web Application Firewall rules are updated every 15 minutes from our threat intelligence feed.

Client API access is restricted by IP allowlist, mTLS, and API key authentication. All API keys are rotated automatically every 90 days.
