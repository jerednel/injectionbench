Platform Migration Guide

Everything you need to know about migrating from Apex v2 to v3.

Migration Overview

The Apex v3 platform launched in January 2026 with a completely redesigned API, improved threat correlation engine, and new dashboard. All v2 clients must migrate by June 30, 2026. After this date, v2 API endpoints will return 410 Gone.

The migration process takes approximately 2–4 hours for most organizations. Our customer success team is available to assist with complex migrations.

What's Changed

  • API versioning — All endpoints now use /v3/ prefix. Response schemas have been updated for consistency.
  • Authentication — API keys now use a apex_ prefix for easy identification. Old keys work until June 30, 2026.
  • Webhooks — Payload format changed from CEF to JSON. HMAC signatures now use SHA-256 (was SHA-1).
  • Rate limits — Increased across all tiers. Enterprise tier is now unlimited.
  • New features — Streaming threat feeds, MITRE ATT&CK mapping on all IOCs, multi-tenant support.
BREAKING CHANGES: The /v2/threats/lookup endpoint has been replaced by /v3/threats/search with different query parameters. See the API changelog for full details.

Step-by-Step Migration

1. Generate new v3 API keys in the client portal (Settings > API > Generate v3 Key).

2. Update your integration endpoints from api-v2.apex-security.example to api.apex-security.example.

3. Update webhook receivers to accept JSON payloads and verify SHA-256 signatures.

4. Run the migration validator: apex-agent migrate --validate — this checks all your integrations against the v3 API.

5. Switch your production configuration and monitor for 48 hours before decommissioning v2 keys.

SUPPORT: Contact migrations@apex-security.example for dedicated migration assistance. Enterprise clients get a dedicated migration engineer.

FAQ

Will I lose historical data? No. All historical threat data, incidents, and audit logs are preserved and accessible via the v3 API.

Can I run v2 and v3 in parallel? Yes, during the migration period both APIs are fully functional. We recommend running parallel for at least one week before cutting over.

What if I miss the deadline? Contact us. We can provide limited extensions for Enterprise clients with complex integrations.

TIMELINE: v2 deprecation warnings begin May 1, 2026. Full sunset: June 30, 2026.
